01 — Problem Statement

What We're Solving

Existing AI assistants are cloud-locked, monolithic, and lack true local identity verification or real-time voice.

Core Gaps in Traditional AI Assistants

  • No biometric identity gate — anyone can use it
  • Voice is either browser-only or laggy HTTP polling, not real-time WebRTC
  • No local system control (power management, browser automation)
  • Cannot execute enterprise workflows (n8n integrations missing)
  • Monolithic architecture — one crash kills everything
  • Real-time web search not integrated into the voice pipeline
  • No decoupled microservice separation for scale and resilience
02 — Architecture

Voice Pipeline Flow

Every spoken word travels through a precisely orchestrated pipeline from camera to speaker.

STEP 1
Face Auth
YOLOv8n → Haar Cascade → LBPH Recognizer

Camera captures frame. YOLOv8n detects person body. Haar cascade refines to face ROI. LBPH recognizes against trainer.yml. Majority vote over 20 frames. Identity confirmed as nnssprasad.

↓
STEP 2
Voice Capture
Microphone → LiveKit WebRTC Room → Audio Stream

After auth, LiveKit Python agent joins the WebRTC room. Audio is streamed in real-time over WebRTC — zero HTTP polling, sub-100ms latency.

↓
STEP 3
STT
Groq Whisper-large-v3 (OpenAI-compatible endpoint)

Audio chunks are sent to Groq's ultra-fast Whisper endpoint. Transcription returns in <300ms. Far faster than OpenAI's own endpoint.

↓
STEP 4
NLU / LLM
Gemini 2.0 Flash (primary) → Groq LLaMA 3.3 (fallback)

Transcript fed to LLM with a system prompt aware of all MCP tools. LLM decides: search web, run local automation, fetch schedule, or trigger n8n workflow.

↓
STEP 5
Tool Exec
MCP Server (port 3143) — FastMCP + Playwright + SerpAPI + powercfg

LLM calls a tool by name. MCP server executes: SerpAPI search, headless Playwright browser action, Windows power plan switch, or n8n webhook POST.

↓
STEP 6
TTS
Google Cloud TTS — en-US-Journey-D Neural voice

Response text synthesized into high-quality audio. Streamed back through LiveKit WebRTC room to the user's speakers. No file writes, no delays.

03 — Microservices

Decoupled Services

Each service is independently startable, testable, and replaceable. One crash does not affect others.

🧠

Brain — Bun Daemon

TypeScript/Bun WebSocket server on port 3142. Manages LLM sessions, task board, conversation history, SQLite DB, n8n wiring, and serves the React dashboard.

bun run src/daemon/index.ts
👁️

Vision Service

YOLOv8n detects persons. Haar cascade refines to face. LBPH authenticates against trainer.yml. OCR via Tesseract. Returns authenticated user or denies access.

vision_service.py
🎙️

LiveKit Voice Agent

WebRTC agent using livekit-agents v1.x. Groq Whisper STT → Gemini 2.0 LLM → Google Cloud TTS. Silero VAD pre-loaded in worker. Speaks greeting on join.

livekit_agent.py dev
⚙️

MCP Tool Server

FastMCP server exposing 5 tools: SerpAPI search, Playwright browser automation, Power Saving Mode, schedule reader, and n8n workflow trigger.

mcp_server.py (port 3143)
🔗

Sidecar Bridge

Orchestrates face auth gate then bridges legacy WebSocket mic/speaker loop to the Bun Brain. Handles auto-reconnect with ping keepalive.

sidecar.py
🔄

n8n Workflows

Runs on localhost:5678. Triggered via webhook by the MCP server. Handles complex multi-step automations: email sends, calendar updates, Slack messages.

n8n (external)

05 — How to Run

One Command, Everything

A single unified launcher starts all 4 microservices with color-coded labeled output.

Step 1 — Install Dependencies (once)
$ cd jarvis-main
$ bun install
$ bun run install:sidecar
# installs pip packages + playwright chromium
Step 2 — Launch Everything
$ bun run launch
Starts in order with staggered delays:
■ BRAIN
■ MCP → port 3143
■ LIVEKIT → LiveKit Cloud
■ SIDECAR → Face Auth + Mic
